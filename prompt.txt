area 7 — Cliente MCP (Unity/Blender) integrado en Gateway
Contexto
El Gateway no debe duplicar lógica MCP: debe invocar al adaptador MCP existente para Unity/Blender.

Objetivo
Crear MCPClient que importe y use las funciones del adaptador MCP (misma venv), con helpers comunes: getSceneHierarchy, captureScreenshot, exportFBX, instantiatePrefab, etc.

Entregables

app/services/mcp_client.py: implementar llamadas reales.
Cargar endpoints/credenciales desde config/settings.yaml.
Reconexión simple para sockets.
Restricciones y consideraciones

No bloquear el event loop: utilizar anyio.to_thread.run_sync si las llamadas son síncronas.
Mapear errores a ErrorEnvelope coherente.
Criterios de aceptación (DoD)

Gateway logra invocar MCP real y devolver datos.
getSceneHierarchy y captureScreenshot operativos.
Buenas prácticas y notas para el agente

Reutilizar constantes/código del MCP si existen.
Backoff en reconexiones; logs claros de fallo.
