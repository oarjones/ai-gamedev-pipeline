Sustituir CLI “eco” por Agente CLI real + configuración por proyecto 

Contexto: El AgentRunner arranca un “echo”. Necesitamos lanzar el CLI del agente real (p. ej. unified_adapter o script Python) en la carpeta del proyecto con env/config correctos. 
 Objetivo: Reemplazar _build_command() y start() para leer la configuración del proyecto y arrancar el binario real. 
 Alcance y tareas: 

Añadir project.json enriquecido (o gateway/config/projects/{id}.yaml) con: ruta del CLI, args, variables de entorno (API keys, URL MCP, modelo IA), timeouts. 

AgentRunner._build_command() debe construir [executable, ...args] desde esa config; soportar Windows/Linux/macOS. 

start(cwd) valida binario y permisos; crea env combinando os.environ + env del proyecto. 

Registrar en logs la línea de comando efectiva (sin secretos). 
 Entradas/Salidas: Entrada: projectId; Salida: status() con running, pid, cwd. 
 Aceptación: 

POST /api/v1/agent/start?projectId=... levanta el CLI real; status muestra running=true. 

Error claro si falta binario/config. 
 Notas & Buenas prácticas: No loguees secretos; usa shlex.quote en logs; maneja rutas con Path.resolve(); añade tests unitarios de construcción de comando
